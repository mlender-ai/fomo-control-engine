# FOMO Control Engine — 에이전트 작업 규약

> 이 파일은 이 저장소에서 작업하는 모든 AI 에이전트(Claude Code·Codex·Cursor·Gemini 등)가 먼저 따른다.

## 🔴 최우선 불변 규칙 — 완료 작업은 무조건 머지한다

**세션에서 만든 작업물은 예외 없이 커밋하고 `origin/main`에 반영한다. 커밋·머지 안 된 채로 세션을 끝내지 않는다.**

과거 사건: WO 구현이 워킹트리에 미커밋으로, 또는 codex 브랜치에 미머지로 방치돼 "됐다"고 착각한 채 사라진 일이 반복됨(WO-70/71/72). 다시는 반복 금지.

작업 종료 전 **반드시** 아래를 실행하고 결과를 확인한다:

1. `git status` — 미커밋 변경(추적/미추적 모두)이 남아 있으면 커밋한다. `git status --short`가 비어야 한다.
2. `git branch` + `git branch -a` — 이 세션이 만든/받은 브랜치에 미머지 커밋이 있으면 main에 통합한다. 병합 끝난 브랜치는 삭제한다.
3. `git log origin/main..main` — 로컬이 원격보다 앞서 있으면(ahead) `git push origin main` 한다. ahead가 0이어야 종료.

검증 게이트(머지 전 필수): `cd backend && python3 -m pytest -q -m "not live"` 통과 + `cd dashboard && npx tsc --noEmit` 0 에러. 실패하면 고치고 나서 머지한다(실패한 채 머지 금지).

예외: 파괴적/되돌리기 어려운 작업이거나 사용자가 명시적으로 보류를 지시한 경우만 미머지 허용. 그 외에는 무조건 머지가 기본값이다.

## 머지 방식
- 이 저장소는 **main 직접 커밋·푸시** 방식으로 운영해 왔다(PR 게이트 없음). 병렬 에이전트 브랜치(`codex/*`)는 작업 후 main으로 통합하고 브랜치를 정리한다.
- 같은 WO를 두 에이전트가 각자 구현해 충돌하면, 정본 하나를 정하고 나머지의 유용한 부분만 흡수한 뒤 중복 구현을 제거한다(두 시스템 공존 방치 금지).

## 작업 산출물
- WO 구현은 관련 `docs/*.md` 갱신을 동반한다.
- 승격/자동적용 등 안전 관련 규정(제안-거부권 등)은 해당 WO 명세를 우선한다.
