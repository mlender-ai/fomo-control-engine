# FOMO Control Engine — 에이전트 헌법

> 이 저장소에서 작업하는 **모든 AI 에이전트**(Claude Code·Codex·Cursor·Gemini 등)는 작업 시작 전에 이 파일을 따른다.
> 정본 우선순위: `docs/PRD.md` > `AGENTS.md` > `HARNESS.md` > 개별 `docs/*.md`.
> 원칙: **Stop Prompting, Start Governing** — 규칙은 프롬프트가 아니라 게이트(CI·훅)로 강제한다.

---

## 🔴 불변 규칙 1 — 완료 작업은 무조건 머지한다

**세션에서 만든 작업물은 예외 없이 커밋하고 `origin/main`에 반영한다. 커밋·머지 안 된 채로 세션을 끝내지 않는다.**

과거 사건: WO 구현이 워킹트리 미커밋·codex 브랜치 미머지로 방치돼 "됐다"고 착각한 채 유실됨(WO-70/71/72). 재발 금지.

작업 종료 전 **반드시** 실행·확인:
1. `git status --short` — 비어야 한다(미추적 포함 전부 커밋).
2. `git branch -a` — 세션이 만든/받은 브랜치의 미머지 커밋을 main에 통합, 병합 끝난 브랜치 삭제.
3. `git rev-list --count origin/main..main` — **0**이어야 한다(푸시 완료).

예외는 단 둘: 파괴적/비가역 작업, 사용자의 명시적 보류 지시. 보류 시 `touch .git/FCE_STOP_OK`로 종료 훅을 1회 통과시킨다(다음 세션 시작 시 삭제).

## 🔴 불변 규칙 2 — WO 착수 전 중복 구현 검사

같은 WO를 두 에이전트가 각자 구현해 충돌한 사건(71이 2벌) 재발 금지. **WO 착수 전 반드시**:
1. `git branch -a` — 같은 WO 번호/주제의 브랜치가 있는지.
2. `git log --oneline -15` + `git status --short` — 이미 구현/진행 중인지.
3. `grep -r "WO-<번호>" docs/ backend/ dashboard/` — 기존 산출물 검색.

이미 있으면: 새로 만들지 말고 그 작업을 **완성·머지**한다. 두 구현이 이미 병존하면 정본 하나를 정하고 나머지는 유용한 부분만 흡수 후 제거(두 시스템 공존 방치 금지).

## 🔴 불변 규칙 3 — 게이트 통과 없이 push 금지

로컬에서 `HARNESS.md`의 게이트를 통과한 뒤에만 push 한다. push 후 **CI 결과를 확인**한다(`gh run list --branch main --limit 1` — failure 방치 금지). CI가 빨간 상태로 세션을 끝내지 않는다.

> 교훈(2026-07-14): ci.yml YAML 오타 하나로 CI가 5일간 0초 실패했고, 아무도 몰랐다. 그 사이 mypy 부채 44→174. **"CI가 있다"와 "CI가 살아있다"는 다르다.**

## 도메인 불변 (트레이딩 엔진 안전 규정 — 코드보다 우선)

- **자동 승격 금지**: candidate→validated 승격은 제안-거부권(veto window) 경유만. 어떤 경로로도 자동 적용하지 않는다.
- **룩어헤드 금지**: 감지기·채점은 확정 캔들만 사용. 새 감지기는 룩어헤드 감사 통과 전 채점 금지.
- **정직한 표본**: 백테스트(사전확률)와 라이브(검증) 출처 분리, N·CI 명시, 표본 부족 시 판정 유보.
- **미검증=관측만**: 미검증 시그니처·고래는 화면 표시만, 방향 판정·자동 진입에 사용 금지.
- **실주문 없음**: 페이퍼는 read-only. 실계좌 주문을 실행하는 코드를 만들지 않는다.
- **적용된 마이그레이션 불변**: `backend/app/db/migrations/*.sql` 기존 파일 수정 금지 — 새 번호로 추가.

## 코드 규율 — 게으름 사다리

새 코드 작성 전 순서대로 자문, 멈추는 첫 칸에서 멈춤: ① 존재할 필요 있나(YAGNI) ② 코드베이스에 이미 있나(재사용) ③ 표준 라이브러리 ④ 깔린 의존성 ⑤ 한 줄이면 한 줄 ⑥ 그제서야 최소 구현. 단 **게으르되 부주의 금지** — 도메인 불변·검증·에러 처리는 잘라내지 않는다.

## 멀티 에이전트 핸드오프 (Claude ↔ Codex)

작업을 넘기거나 세션을 마칠 때 마지막 출력(커밋 본문·이슈 코멘트 등)에 HANDOFF 블록을 남긴다:

```
HANDOFF
- 목표: <한 줄>
- 한 일: <건드린 파일·핵심 변경>
- 안 한 일/막힌 곳: <남은 작업·블로커>
- 다음 액션: <다음 에이전트가 바로 할 일 1~3개>
- 검증: <통과/실패한 게이트>
- 머지: <origin/main 반영 여부 — 불변 규칙 1>
```

## 작업 산출물 규약

- 새 WO는 `docs/templates/WO_TEMPLATE.md` 형식으로 고정 후 구현.
- WO 구현은 관련 `docs/*.md` 갱신을 동반한다(코드만 있고 문서 없는 기능 금지).
- 이 저장소는 **main 직접 커밋·푸시** 방식(PR 게이트 없음). 커밋 메시지에 WO 번호 명시.

## 시크릿

`.env`·`*.pem/key` 등은 편집 금지(Claude는 `.claude/hooks/protect-secrets.sh`가 결정론 차단). 변경이 필요하면 `.env.example`에 반영하고 실제 값은 사용자가 직접 관리.
