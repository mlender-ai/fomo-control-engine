# 라이브 포지션 UI 전수조사

기준 WO: `WO-FCE-LIVE-UI-HIERARCHY-01`  
검증일: 2026-07-19  
대상: 라이브 포지션 미니멀/프로, 주식 기초자산 조인, 모바일 390px

## 결론

첫 화면을 `대상 선택 → 소스 요약 → 판정/차트` 순서로 고정했다. 미니멀은 요약만,
프로는 핵심 6개 레이어만 기본 노출하고 나머지 근거는 접힘 영역에서 보존한다.
데이터·판정·주문 경로는 바꾸지 않았다.

## 실측 기준

- 1440×900 미니멀: 워크스페이스 y=334.6, 고래 요약 60px, 가로 overflow 0.
- 390×844 미니멀: 툴바 111.3px, 워크스페이스 y=386.6, 가로 overflow 0.
- 1440×900 프로: 차트/액션 플랜 시작 y=556.9, 기본 레이어 6개, 고급 레이어 기본 접힘.
- PWA manifest·192/512 아이콘·same-origin API proxy·14개 제품 라우트 자산 2xx 확인.

## 유지 규칙

1. 미니멀에 새 상세 데이터를 직접 추가하지 않는다. 기존 요약과 교환하거나 프로 접힘으로 보낸다.
2. 같은 숫자는 한 컴포넌트만 소유한다. 소스 상태 배너는 가격/베이시스를 복제하지 않는다.
3. 프로 기본 레이어는 6개를 상한으로 한다. 추가 레이어는 고급 그룹에 둔다.
4. 비동기 UI 삭제/해제는 진행 중인 조회 응답이 되살리지 못하게 mutation version을 둔다.
5. 390/1440 실측과 Playwright 시각 기준선을 함께 갱신하지 못하면 UI 변경을 완료로 보지 않는다.
