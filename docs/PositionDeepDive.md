# 포지션 심화 판정 계약

## 목적

라이브 포지션의 우측 판정 계기판은 매매 지시가 아니라 서로 다른 데이터 출처의 동시 관측 기록이다. 검증된 주식 기초자산 선물에만 Bitget 퍼페추얼, Toss 기초자산, 내 포지션을 조인한다. BTC·ETH 같은 순수 크립토에는 Toss를 연결하지 않는다.

## 진입 스냅샷

`position_entry_snapshot`은 포지션별 한 번만 생성한다. 포지션 생성 당시 원본 데이터가 없는 기존 포지션은 현재 값을 과거 진입값처럼 저장하지 않고 `capture_policy=first_observed_proxy`와 실제 `captured_at`을 기록한다. 사용자가 나중에 논거를 입력해도 최초 자동 스냅샷은 보존하고 현재 비교의 논거만 갱신한다.

## 교차신호 계약

- 모든 항목의 `sources` 길이는 2 이상이다.
- `status=active`는 필요한 실제 값이 모두 있을 때만 허용한다.
- `status=unavailable`은 값·시장 세션·공급 범위 중 하나가 없을 때 사용한다.
- `moat_reason`은 단일 소스로 같은 관측을 만들 수 없는 이유다.
- 실현 청산 히트맵은 미래 청산 예상치가 아니며 `truth_label`과 표본 N을 그대로 전달한다.
- 베이시스 과거열은 Bitget 4시간 분석 캔들과 해당 시각 이전 Toss 조정 일봉 종가를 정렬한 관측값이다. 마지막 현재가는 `live_observation`으로 구분한다.
- 미국 주식 투자자별 수급이 제공되지 않거나 시장이 비거래 시간이면 수급 신호를 비활성으로 둔다.

## 레버리지 중첩

SOXL은 검증된 기초자산 메타데이터의 3배와 내 Bitget 퍼페추얼 배율을 곱해 명목 실효 익스포저를 표시한다. 이는 실제 포트폴리오 베타나 청산 확률이 아니다. 감쇠 수치는 Toss 조정 일봉의 최근 변동성으로 기초 분산을 근사한 20거래일 수치이며 방법과 경고를 함께 표시한다.

## 판정 결과

포지션 교차판정은 기존 `scout_judgment_snapshots`와 `scout_judgment_outcomes`를 공유한다. 같은 포지션은 하루 한 번 최초 관측을 저장한다. 작업자가 실제로 경과한 1·5·20일 이후 현재 Bitget 가격을 기록한다. 포지션이 먼저 종료돼도 미기록 시점이 도래한 심볼만 Bitget 공개 시세를 읽어 결과를 이어간다. N<30은 항상 `sample_low=true`다. 과거 포지션 시점이나 아직 경과하지 않은 결과는 채우지 않는다.

전체 교차판정 성적과 함께 방향 해석이 가능한 신호 유형별 분포를 분리한다. 음(-) 베이시스 폭 확대, 비영(非零) 펀딩과 기초 모멘텀, 실제 제공된 기초 수급만 방향 결과를 채점한다. 청산 선반 상대 위치와 레버리지 중첩처럼 자체 방향 예측이 없는 관측은 적중률을 만들지 않는다.

## 안전 경계

- 판정·시뮬레이션은 주문 API를 호출하지 않는다.
- 부분 축소 계산은 진입가·증거금 배분·청산가가 고정된 정적 가정이다.
- 교차신호는 Entry Score나 자동 진입 근거로 사용하지 않는다.
- 읽기 전환선은 확정 캔들 구조의 재평가 조건이며 행동 지시가 아니다.
