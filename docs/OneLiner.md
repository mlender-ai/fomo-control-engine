# OneLiner — TA별 1줄 판정 엔진 (WO-FCE-43 Part A)

"오를 확률이 높아? 내릴 확률이 높아? TA마다 딱 1줄." 각 TA 모듈의 내부 상태를
고정 어휘 1줄로 증류한다. 모듈 간 충돌은 **은폐하지 않고 그대로 노출**하며,
판단은 사용자 몫이다.

구현: `app/analyst/oneliner.py` · 발행: `chart_analysis.one_liners` (스카우트/포지션 공통)

## 출력 형식

```json
{
  "lines": [{ "module", "module_label", "stance", "phrase", "confidence_class", "evidence_ref" }, ...7개],
  "counts": { "상방": n, "하방": n, "횡보": n, "판단불가": n },
  "overall_stance": "상방|하방|횡보|판단불가",
  "summary": "종합: 상방 4 · 하방 2 · 판단불가 1"
}
```

- `stance` ∈ {상방, 하방, 횡보, 판단불가} · `confidence_class` ∈ {강, 중, 약} · `phrase` ≤ 25자
- **자유 문장 생성 금지**: phrase는 어휘 사전(`PHRASES`)의 조합만 허용. 사전 밖 phrase로
  빌드하면 `OneLinerVocabularyError`. stance는 phrase에서 파생되므로 문구-방향 불일치가
  구조적으로 불가능하다.
- 공용 폴백 `"데이터 부족"`(항상 판단불가)은 모듈 입력 자체가 없을 때만.
- **종합 stance**: 컨플루언스(CI 가중)가 있으면 그것을 사용, 없으면 단순 다수결.
  CI 가중은 종합에만 쓰고 개별 줄은 왜곡하지 않는다.

## 결정 트리 (모듈별)

### 와이코프
```
phase == trending ──────────────→ "추세 진행 중" (stance = 추세 방향: bullish→상방, bearish→하방, 그 외 횡보)
side == accumulation ───────────→ "매집 우세" (상방)
side == distribution ───────────→ "분산 우세" (하방)
그 외 (undetermined 등) ─────────→ "레인지 미형성" (판단불가)
```
- confidence_class: 해당 side 점수 ≥70 강 / ≥55 중 / 그 외 약.
- **혼합 신호 규칙**: side가 판정됐어도 반대측 최고 신뢰 이벤트 ≥ 65면 confidence_class를
  "약"으로 강등한다. 판정은 유지하되 강도로 혼합을 노출 — 판정 뒤집기(은폐)는 하지 않는다.
- 엔진 레벨에서도 `wyckoff.conflict_note`로 같은 사실을 명시한다 (Part B 참조).

### 유동성
```
확정 스윕 존재 ─ 최신(return_at) 스윕 기준:
  sell_side (저점 스윕) ─────────→ "저점 청소 후 반등 구조" (상방)
  buy_side (고점 스윕) ──────────→ "고점 청소 후 하락 경계" (하방)
확정 스윕 없음 + 풀 존재 ────────→ "풀 대기" (횡보)
그 외 ──────────────────────────→ "신호 없음" (판단불가)
```
- 양방향 확정 스윕 공존 시 **최신 스윕 우선** (시장 구조는 진화 — 최신 청소가 유효 신호).
- confidence_class: 스윕 grade Strong→강 / Mid→중 / Weak→약.

### 볼륨
```
실체결 데이터 없음 ──────────────→ "데이터 부족" (판단불가)
delta_ratio ≥ +0.2 ─────────────→ "매수 체결 우위" (상방)
delta_ratio ≤ −0.2 ─────────────→ "매도 체결 우위" (하방)
|delta_ratio| < 0.2 ────────────→ "균형" (횡보)
```
- confidence_class: |delta| ≥0.45 강 / ≥0.3 중 / 그 외 약.

### 하모닉
```
패턴 없음/방향 불명 ─────────────→ "패턴 없음" (판단불가)
최고 신뢰 패턴 bullish ──────────→ "상방 반전 구간 접근" (상방)
최고 신뢰 패턴 bearish ──────────→ "하방 반전 구간 접근" (하방)
```
- confidence_class: 신뢰도 ≥80 강 / ≥70 중 / 그 외 약. PRZ 중심이 현재가에서 5% 초과
  이탈 시 "약"으로 강등 ("접근" 판정의 정직성).

### 레벨
```
지지-저항 밴드 내 현재가 위치:
  하단 1/3 ──────────────────────→ "지지 위 유지" (상방)
  상단 1/3 ──────────────────────→ "저항 아래 정체" (하방)
  중앙 1/3 ──────────────────────→ "레벨 사이 중립" (횡보)
지지 또는 저항 부재 ─────────────→ "데이터 부족" (판단불가)
```
- confidence_class: 근접 레벨 score ≥80 강 / ≥70 중 / 그 외 약.

### 수급 (파생)
```
파생 신호 없음 ──────────────────→ "데이터 부족" (판단불가)
펀딩 extreme & 양수 (롱 과밀) ────→ "롱 쏠림 경계" (하방)
펀딩 extreme & 음수 (숏 과밀) ────→ "숏 쏠림 경계" (상방)
그 외 ──────────────────────────→ "중립" (횡보)
```
- 쏠림은 역방향 경계 신호다 — 롱 쏠림 = 하방 리스크. confidence_class: 쏠림 점수 ≥70 강 / 그 외 중.

### 지표
```
3표 합산: RSI(>55 / <45) + MACD 히스토그램 부호 + 볼린저 중심선 대비 종가
  합 > 0 ────────────────────────→ "상승 우세" (상방)
  합 < 0 ────────────────────────→ "하락 우세" (하방)
  합 = 0 ────────────────────────→ "중립" (횡보)
```
- confidence_class: |합| 3 강 / 2 중 / 1 약.

## 경계 케이스 우선순위 (설계 결정)

1. **와이코프 undetermined + UTAD 확정 공존**: phase가 undetermined면 개별 이벤트가
   확정이어도 1줄은 "레인지 미형성"(판단불가). 이벤트 자체는 마커/근거로 노출되고,
   유동성 모듈이 같은 가격 행동(고점 청소)을 자기 줄로 판정한다 — 모듈 간 역할 분리.
2. **모듈 내 충돌** (양방향 스윕, side 판정 vs 반대 이벤트): 각 모듈의 결정 트리가
   단일 stance를 산출하되 confidence_class 강등으로 혼합을 노출. 모듈 간 충돌은
   종합 카운트에서 그대로 보인다.
3. **판정 불가 vs 횡보 구분**: 데이터/신호 부재 = 판단불가, 신호가 "중립"을 가리킴 = 횡보.

## 금지

- 어휘 사전 밖 phrase 생성 (빌드 실패로 강제)
- 컨플루언스 가중으로 개별 모듈 줄 왜곡
- 모듈 간 stance 불일치 은폐
