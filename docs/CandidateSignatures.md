# Candidate Signatures

WO-FCE-60 후보 시그니처는 검증 전까지 `candidate` 상태다. 백테스트와 판단 원장에는 저장하지만 일반 UI, 알림, 게이지, 브리핑에는 노출하지 않는다. WO-FCE-71의 일일 채점 잡이 심사 현황만 엔진 상태에 공개하며, 검증 기준을 충족해도 자동 승격하지 않고 거부권이 있는 승격 제안을 만든다.

## 공통 규칙

- 모든 감지는 전달된 확정 캔들 prefix만 사용한다.
- fractal swing은 우측 2개 캔들이 생긴 뒤에만 확정한다.
- 백테스트 가상 진입은 이벤트 확정 캔들의 종가다.
- 후보 통계는 공개 `historical_backtest.stats`에서 제외한다.
- `score_candidates`는 매일 저우선으로 전 보유·추적 심볼의 리플레이를 갱신한다.
- 백테스트는 사전확률, 확정 이후 라이브 원장 채점은 검증 표본으로 분리 기록한다.
- 승격 기준은 전체와 충분한 레짐 표본에서 `N >= 30`, `win@1R CI 하한 >= 50%`다.
- 기준 충족은 `signature_promotion` 제안만 생성한다. 승인 전 상태는 계속 `candidate`다.
- 룩어헤드 감사 실패가 하나라도 있으면 해당 일일 채점 전체를 중단한다.

## FVG (미충전 갭)

상방 갭은 `high[i-2] < low[i]`, 하방 갭은 `low[i-2] > high[i]`다. 갭 구간은 두 값 사이이며 중간값 50%까지 후속 가격이 도달하면 충전된 것으로 본다.

## Order Block (매물 존)

BOS/CHoCH를 몸통 종가로 확정한 뒤, 돌파 직전 마지막 반대색 캔들의 몸통 `[min(open, close), max(open, close)]`을 존으로 저장한다. 돌파 이후 처음 존과 겹친 확정 캔들만 재시험 이벤트다.

## VCP (변동성 수축)

fractal swing 최근 4개로 만든 3개 파동 진폭을 `A1, A2, A3`라 할 때 `A2 <= 0.75*A1`, `A3 <= 0.75*A2`를 모두 만족하고 마지막 캔들 상대거래량이 20캔들 평균의 `0.7` 미만이면 후보로 기록한다.

## Full Alignment

확정 캔들의 비전환 스탠스, HTF 방향, 검증된 모듈 4개 이상의 만장일치가 같은 방향일 때만 후보로 기록한다. 이후 방향 적중을 측정해 늦은 만장일치 신호의 실제 예측력을 검증하며, 표본 또는 CI가 부족하면 `예측력 미검증`으로 표시한다.

## Futures-led Rally

확정된 파생 구간에서 가격 상승, 선물 체결 우위, OI 증가가 나타나지만 현물 유입이 확인되지 않은 에피소드를 숏 방향 되돌림 후보로 기록한다. 미확정 구간과 30일 분포 표본이 부족한 관측은 후보로 만들지 않는다. 표본 또는 CI가 부족하면 `예측력 미검증`으로 표시한다.

## Stage 2 Template

다음을 모두 만족할 때 활성 상태다.

1. `close > MA150 > MA200`
2. `MA200(now) - MA200(20 bars ago) > 0`
3. 종가가 최근 최대 260캔들 고점 대비 25% 이내

이 상태는 이벤트가 아니라 상태 라벨이다. 비활성에서 활성으로 바뀐 확정 캔들만 백테스트 이벤트로 기록한다. 즉시 소비처는 주식·지수 유니버스 발견 게이트뿐이며 기본값은 활성이다.
