# UX Principles

Phase 4 정보 설계 개편(WO-FCE-09~11)의 상위 규범. 이후 모든 UI WO는 아래 4원칙을 통과해야 한다.

## 1. Verdict first, evidence on demand

판단·행동이 최상단, 근거는 클릭/펼침으로.

- 판단의 단일 소유자는 Verdict Bar다: 상태 라벨+건강도, next_action 한 문장, 기준 시각+신선도.
- `next_action`은 백엔드 액션 플랜에서 파생되는 결정론 문장(`headline_action`)이다. Tier 1에 새 정보를 발명하지 않는다 — 전부 기존 필드 파생.
- 근거(와이코프/하모닉/볼륨/지표/리스크/기록)는 기본 접힘 아코디언. AI 해설은 판단이 아니라 근거이므로 액션 플랜 아래 "해설 보기" 접힘에 둔다.

## 2. One glance = one decision

첫 화면(스크롤 없이, 1080p 기준)은 "상태 + 다음 행동 + 핵심 가격 3개"만으로 완결된다.

- 구성: Verdict Bar → 차트(미니멀 레이어) + 액션 플랜.
- 폼, 히스토리, 상세 분해는 첫 화면 아래 아코디언 내부로.
- 차트는 기본 화면에서 한 번에 한 질문만 답한다(WO-42): 미니멀 차트에는 레이어 선택 UI가 없다. 평결의 1순위 근거가 오버레이 1개를 결정하고, TA 다중 렌더는 프로 모드 전용이다.

## 3. No naked jargon

전문용어는 반드시 플레인 라벨과 동행한다 (상세는 WO-11).

- 짧은 한국어 라벨이 1차 표기, 원어/약어는 괄호: "고점 돌파 실패(UTAD)", "반전 후보 구간(PRZ)".
- Verdict Bar의 next_action에는 TA 원어 사용 금지 — plain 언어만.
- snake_case 원문(`balanced_flow` 등)이 화면에 노출되는 경로는 전수 제거한다.
- 라벨 매핑은 `dashboard/lib/labels/`(taGlossary)에 집중, 컴포넌트 하드코딩 금지.

## 4. Every number earns its pixel

같은 숫자를 두 곳 이상에 표시하지 않는다. 표시 위치는 단일 소유자 원칙.

- 건강도 → Verdict Bar 단독. 손익률 → Verdict Bar(목록 카드는 스캔용 예외).
- 진입가/현재가 → 차트 가격선 + 액션 플랜 distance 계산에만 존재. 칩 레일 없음.
- 무효화/익절/감시 가격 → 액션 플랜. 리스크 분해 → 리스크 아코디언.
- 새 숫자를 추가할 때는 기존 소유자를 먼저 확인하고, 있으면 그 자리에서만 갱신한다.

## 5. Minimal mode has a hard budget

WO-FCE-41 이후 기본 화면은 미니멀 모드다. 프로 모드는 기능 보존 영역이고, 기본 화면은 정보 예산을 넘기면 실패로 본다.

- 포지션 평결 카드: 수치 토큰 7개 이하, 버튼 4개 이하, 원어/약어 노출 0.
- 포지션 목록 카드: 수치 토큰 5개 이하. 중복 배지는 숨긴다.
- 스카우트 기본 표: 4컬럼 고정(`심볼`, `기울기`, `근거`, `트리거까지`). 원래 12컬럼 표는 프로 모드에만 둔다.
- 화면당 버튼은 6개 이하를 목표로 한다. 초과가 필요하면 무엇을 뺄지 지시서에 명시해야 한다.
- 신규 기능은 기본적으로 프로 모드에만 추가한다. 미니멀 편입은 "예산 교환"이 명시된 WO에서만 허용한다.
- 미니멀 차트에 `구조`·`전체` 같은 통합 프리셋을 만들지 않는다. 여러 TA를 한 번에 보고 싶으면 [자세히] 프로 모드로 이동한다.
- 미니멀 컴포넌트는 `data-budget-*` 속성을 유지한다. `dashboard/scripts/check-minimal-budget.mjs`가 이 규칙의 최소 회귀 방어선이다.

## 6. Pro mode uses progressive disclosure

프로는 모든 정보를 한 번에 펼치는 화면이 아니라, 더 깊게 확인할 수 있는 화면이다.

- 차트 기본 레이어는 최대 6개다. 나머지는 `고급 레이어`에 접어 기본 판단 흐름과 분리한다.
- 고래 지갑·최근 체결, 파생상품 근거, 추가 관측 가격, 판정 ledger는 기본 접힘이다.
- 소스 연결 배너는 연결 상태만 소유한다. 가격·베이시스·수급 숫자는 차트/근거 패널의 단일 소유자가 표시한다.
- 선택된 근거가 없으면 빈 근거 패널을 만들지 않는다. 빈 상태 설명 때문에 차트가 밀리면 실패다.
- 조회 응답과 사용자 삭제/해제가 경합할 수 있는 목록은 mutation version으로 stale 응답을 차단한다.

## 완료 판정 시나리오 (회귀 기준)

1. 라이브 포지션 클릭 → 3초 내 "상태 + 지금 볼 가격 + 발생 시 행동"을 읽을 수 있다.
2. 와이코프를 모르는 사용자가 화면의 모든 1차 텍스트를 이해할 수 있다.
3. 평결 근거 문장을 클릭하면 차트가 해당 근거 하나만 보여준다.
4. 같은 숫자를 화면에서 두 번 보지 않는다.
5. 390px에서 주요 툴바가 130px를 넘거나 document 가로 overflow가 생기지 않는다.
