# UX Principles

Phase 4 정보 설계 개편(WO-FCE-09~11)의 기준 문서. 신규 화면·컴포넌트는 아래 5원칙을 통과해야 한다.

## 1. Verdict first, evidence on demand

판단(상태 + 지금 볼 가격 + 발생 시 행동)이 항상 화면 최상단에 온다.
근거(와이코프, 하모닉, 기술지표, 수급, 리스크 분해)는 기본 접힘 상태의 아코디언으로 내려가고, 사용자가 요청할 때만 펼쳐진다.

- 판단의 단일 소유자는 Verdict Bar다. 같은 판단 문구/상태 라벨을 다른 패널에서 반복하지 않는다.
- AI 해설은 판단이 아니라 근거다. 근거 계층에 배치한다.

## 2. One glance = one decision

라이브 포지션을 선택하면 스크롤 없이 첫 화면에서 하나의 결정을 내릴 수 있어야 한다.

- 첫 화면 구성: Verdict Bar → 차트 + 액션 플랜. 이 안에서 "유지/축소/이탈 판단에 필요한 최소 정보"가 완결된다.
- 폼, 히스토리, 상세 분해는 첫 화면 아래로 내린다.

## 3. One chart, one story

차트는 한 번에 하나의 분석 이야기만 렌더링한다.

- TA 레이어는 상호 배타 토글이다: 기본(미니멀) / 지지·저항 / 와이코프 / 하모닉 / 수급.
- 기본 레이어는 캔들 + 거래량 + 핵심 가격(진입/현재/청산/무효화 + 1차 지지/저항)만 보여준다.
- 근거 아코디언의 각 TA 섹션에서 클릭 1번으로 해당 레이어 단독 뷰로 전환할 수 있어야 한다.

## 4. No naked jargon

1차 텍스트(버튼, 라벨, 상태, 액션)는 와이코프를 모르는 사용자도 읽을 수 있어야 한다.

- 짧은 한국어 라벨을 1차로 쓰고, 원어/약어는 괄호로 뒤에 붙인다. 예: "반전 후보 구간(PRZ)", "누적 수급(CVD)".
- 원어 단독 노출은 근거 계층의 상세 텍스트에서만 허용한다.
- 라벨 매핑은 `dashboard/lib/labels/`(taGlossary)에 집중시키고 컴포넌트에 하드코딩하지 않는다.

## 5. Every number earns its pixel

모든 숫자는 화면에서 단일 소유자를 가진다. 같은 숫자를 두 번 보여주려면 두 번째 자리를 지워야 한다.

- 건강도 → Verdict Bar. 손익률/진입가/현재가/청산가/청산가 거리 → Verdict Bar 메트릭 행.
- 무효화/익절/감시 가격 → 액션 플랜. 리스크 분해 점수 → 리스크 근거 섹션.
- 새 숫자를 추가할 때는 먼저 기존 소유자가 있는지 확인하고, 있으면 그 자리에서만 갱신한다.

## 완료 판정 시나리오 (회귀 기준)

1. 라이브 포지션 클릭 → 3초 내 "상태 + 지금 볼 가격 + 발생 시 행동"을 읽을 수 있다.
2. 와이코프를 모르는 사용자가 화면의 모든 1차 텍스트를 이해할 수 있다.
3. 와이코프만 보고 싶을 때 클릭 1번으로 차트가 와이코프 단독 뷰가 된다.
4. 같은 숫자를 화면에서 두 번 보지 않는다.
