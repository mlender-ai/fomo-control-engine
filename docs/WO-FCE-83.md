## 진단 (코드 확정)
- 워커 `paper_engine`은 90초마다 성공했지만 벤치마크 시드 ETH 거래가 TP1 전 `take_profit_pressure`로 잘못 청산됐다.
- `backend/app/paper/service.py`의 `_bootstrap_validation_positions`는 정책 무효 거래도 벤치마크 거래로 계산해 이후 시드를 영구 차단했다.
- 벤치마크 시드는 처리 시각에 진입하지만 청산은 캔들 시작 시각을 기록해 `exit_at < entry_at`이 될 수 있었다.

## 작업
### 1. 정책 무효 시드 복구
- `policy_invalid:*` 태그 거래는 시드 완료로 계산하지 않는다.
- 동일 캔들 재진입은 금지하고 다음 새 확정 캔들에서만 기존 성적·체크리스트·신선도 게이트를 그대로 적용한다.
- 복구 진입은 `validation_bootstrap_recovery`와 무효 시드 수를 기록한다.

### 2. 원장 시각 정합
- 부분익절·청산·보유 갱신 시각은 진입 시각보다 과거가 될 수 없게 한다.

## 수용 기준
- [x] 정책 무효 시드 뒤 새 확정 캔들에서 검증 포지션이 복구된다.
- [x] 같은 캔들에서는 복구 진입이 생성되지 않는다.
- [x] 유효한 벤치마크 거래가 있으면 추가 시드를 만들지 않는다.
- [x] `exit_at >= entry_at`이 항상 성립한다.
