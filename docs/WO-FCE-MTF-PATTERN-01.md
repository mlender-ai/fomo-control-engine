## 킬존 대조와 위생 필요성

`docs/FCE-MOAT-STRATEGY-01.md` §2의 차트 기능 경쟁은 하지 않는다. 이 작업은 신규 감지기나 장식 레이어가 아니다. 현재 4시간봉 단일 분석을 전체 시장 구조처럼 표시하고, 일봉은 캔들 부족으로 실패하며, 미확정 진행 봉까지 감지 입력에 들어가는 **데이터 오표시와 관측 불능**을 바로잡는 위생 작업이다.

## 진단 (코드 확정)
- `dashboard/components/live-position-cockpit.tsx`의 메인 관제 차트 호출은 `4h`로 고정되어 일봉·1시간·분봉을 요청하지 않는다.
- `backend/app/exchange/bitget/provider.py`의 일반 캔들 응답은 일봉에서 분석 하한 100개를 충족하지 못할 수 있지만 히스토리 백필로 보완하지 않는다.
- `backend/app/positions/chart_analysis.py`는 현재 진행 중인 마지막 캔들을 제거하지 않고 와이코프·하모닉 엔진에 전달한다.
- 기존 `wyckoff_mtf`는 같은 캔들 집합을 내부 집계한 값으로, Bitget에서 각 시간봉을 독립 조회한 결과가 아니다.

## 작업

### 1. 확정 캔들 경계 강제
- 시간봉 길이보다 덜 지난 마지막 캔들을 구조·패턴·레벨 계산에서 제외한다.
- 응답의 데이터 품질에 제외 개수와 마지막 확정 시각을 명시한다.

### 2. 일봉 백필
- 일반 Bitget 캔들이 분석 하한에 못 미치면 공개 history-candles 경로로 확정 캔들 200개를 백필한다.
- 백필 실패는 다른 시간봉의 결과까지 실패시키지 않고 해당 시간봉만 `unavailable`로 기록한다.

### 3. 다중 시간봉 패턴 매트릭스
- 기존 와이코프·하모닉 엔진을 그대로 재사용해 `1D / 12H / 4H / 1H / 15M`을 독립 분석한다.
- 각 행에 캔들 수, 마지막 확정 시각, 와이코프 레인지·이벤트 수, 하모닉 패턴 수와 최고 패턴을 반환한다.
- 임계값 완화나 후보 패턴 승격은 하지 않는다.

### 4. 메인 프로 관제 시간봉 전환
- 프로 차트 위에 시간봉별 감지 상태를 한 줄 매트릭스로 표시한다.
- 시간봉을 누르면 그 시간봉의 실제 캔들 차트와 마커로 전환한다. 다른 시간봉의 마커를 현재 차트 위에 투영하지 않는다.
- 미검출 문구는 현재 시간봉을 명시해 전체 시간봉 미검출로 오해하지 않게 한다.
