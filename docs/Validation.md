# Validation — 자율 검증 루프 (WO-FCE-37)

사용자 목표 "자율적으로 검증하고 성장하는 엔진"의 구현부. WO-36이 숫자를 통계적으로
정직하게 만든 위에서, 이 루프는 **나빠진 신호를 스스로 강등·격리하되 좋아지는 방향은
전부 사람 승인을 거치게** 한다.

## 핵심 원칙 — 비대칭 자율 (Asymmetric Autonomy)

엔진의 자율 권한은 방향에 따라 다르다:

- **강등·격리·보수화 = 자율 실행** (즉시, 통보만) — 나빠진 신호를 계속 쓰는 것이 최대 리스크.
- **승격·완화·복귀 = 제안-승인 필수** (WO-22 불변).

이 비대칭이 "스스로 성장하되 폭주 불가능한" 구조의 전부다. 성장 = 표본 축적 → CI 수축 →
게이트 통과 시그니처 증가 + 부패 시그니처 자동 퇴출. 폭주 경로 없음.

## 시그니처 상태 머신 (`analyst/signature_registry.py`)

```
candidate → validated → degraded → quarantined
                ↑___________(복귀: 제안-승인)__________|
```

| 상태 | 조건 | 게이트/가중/브리핑 |
|---|---|---|
| `candidate` | CI 하한 < 50% 또는 N < 30 | 브리핑 표기만, 게이트·가중 불참 |
| `validated` | WO-36 게이트 통과 (CI 하한 ≥ 50%, N ≥ 30) | 발견 자격 + 가중 참여 |
| `degraded` | 부패 감지 (아래 규칙) | 발견 게이트 제외(자율), 브리핑 "성능 저하 관찰 중" |
| `quarantined` | 부패 지속 | 브리핑 증거 제외(자율), **데이터 수집은 지속** (복귀 판정용) |

- 강등·격리 전이는 자율 즉시 + 주간 리포트 통보.
- 복귀(quarantined/degraded → validated)는 `recover_proposed` 로 제안만 생성되고,
  `POST /api/review/signatures/{key}/recover` 승인 후에만 `recover_applied` 로 적용.
- 근거 스냅샷 없는 전이는 `record_transition` 이 거부한다 (감사 무결성).

## 부패 감지 규칙 (`validation/decay.py`, 주 1회)

입력: WO-36 워크포워드 곡선 + OOS 플래그 + (있으면) 라이브 원장. 전부 결정론.

- **R1** 최근 충분표본 창의 승률 **CI 상한 < 장기 승률 점추정** → degrade 신호
- **R2** OOS unstable (학습/검증 괴리 > 15%p) → degrade 신호
- **R3** 라이브 vs 백테스트 괴리 > `FCE_DECAY_LIVE_DIVERGENCE_PCT`(20%p), 라이브 N ≥ `FCE_DECAY_LIVE_MIN_SAMPLE`(15) → degrade 신호
- **R4** `degraded` 상태에서 부패 **재관측** → `quarantined`. 재관측 = 강등 이후 통계가
  재생성된(새 데이터) 상태에서 부패 신호 지속. 같은 관측으로 스윕을 반복 실행해도
  격리로 점프하지 않는다 (`stale_skipped`).
- **R5** `degraded`/`quarantined` 에서 최근 2창 연속 게이트 재통과 (CI 하한 ≥ 50%) → **복귀 제안** (자율 아님)
- **레짐별 부분 부패**: 전체는 멀쩡하지만 특정 레짐 슬라이스만 CI 상한이 전체 점추정보다
  낮으면 degrade에 `regime` 태그를 남긴다. 게이트 제외는 보수적으로 전 레짐에 적용된다
  (레짐 한정 제외는 게이트에 레짐 차원이 생기면 도입). 전면 부패 신호(R1~R3)에는
  레짐 태그를 붙이지 않는다.
- **부패 판정의 대표 통계**: signature_key는 심볼 무관이므로, 심볼별 행 중 **표본이 가장 큰
  행**을 대표로 쓴다 (방금 분석된 소표본 알트가 대표본 통계를 밀어내는 것을 방지).

## 주간 셀프 오딧 (`build_self_audit`)

WO-22 주간 리포트에 2단으로 첨부되어 자율과 승인의 경계를 매주 가시화한다:

- **엔진이 이번 주 스스로 한 일 (자율)**: 강등·격리 전이 목록 + 근거 수치
- **사용자 승인 대기 중인 일 (제안)**: 복귀 제안, 상한/킬스위치로 이월된 제안
- 부패 관찰 중(`degraded`)·격리(`quarantined`) 시그니처 목록
- 라이브-백테스트 괴리 상위 3
- 메타 무결성: 자율 강등 오판율

## 메타 무결성 — 자기 검증의 검증 (`autonomy_scorecard`)

`autonomy_log` 에 전이·근거 스냅샷·당시 통계를 전수 기록한다. 자율 강등/격리 후 복귀
승인된 비율(`misjudgment_rate_pct`)을 집계 — **자율 규칙 자체도 채점 대상**이다.
"이 강등이 옳았나"를 사후 확인할 수 있다.

## 안전장치

- **주간 전이 상한** `FCE_AUTONOMY_WEEKLY_TRANSITION_CAP`(기본 5) — 초과분은 자율 실행 대신
  제안으로 이월 (통계 시스템 버그로 인한 대량 격리 방어).
- **전면 격리 방어**: 모든 시그니처가 `quarantined` 도달 시 (엔진 전면 불신) `critical` 플래그 +
  신규 자율 강등 동결(`frozen`).
- **킬스위치** `FCE_AUTONOMY_ENABLED=false` — 전 전이가 제안 모드로 (자율 상태 변경 없음).

## 금지

- 승격·완화 방향 자율 실행 금지 (예외 없음).
- 근거 스냅샷 없는 상태 전이 금지.
- `quarantined` 시그니처의 데이터 수집 중단 금지 (복귀 판정 불능 방지).
