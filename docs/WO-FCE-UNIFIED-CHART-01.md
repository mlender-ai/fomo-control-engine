# WO-FCE-UNIFIED-CHART-01 — 청산 히트맵 통합 라이브 차트

- 대상: fomo-control-engine (127.0.0.1:8876)
- 담당: Claude Code
- 우선순위: P1 (페이퍼 트레이딩 검증 기간 중 관측 도구 개선)
- 레퍼런스: Coinglass Liquidation Heatmap 스타일 (첨부 영상 2건)

---

## 0. 배경 / 문제 정의

현재 FCE는 관측 정보가 세 군데로 흩어져 있다.

1. **실현 청산 밀집도**: 라이브 포지션 내 별도 카드. 수평 밴드 + 축소된 4H 캔들로 자체 렌더링. 메인 차트와 뷰포트/시간축이 독립적이라 눈으로 대조해야 함.
2. **ETHUSDT 차트**: 플랜·레벨·유동성·볼륨·와이코프·하모닉·EMA·온체인 8개 레이어가 **배타 탭**. 유동성을 보려면 플랜을 꺼야 하고, 하나씩 클릭해서 순회해야 함.
3. **액션 플랜 사이드바**: 익절/감시/무효화 가격이 차트 우측에 텍스트로 있고, 차트에는 클릭 시에만 강조.

**목표 상태**: 하나의 캔버스에서 청산 밀도 히트맵(배경) + 캔들 + 현재가 라인 + 선택 레이어들을 동시 합성. 필터는 차트 상단 인라인 칩으로, 클릭 즉시 같은 캔버스 리렌더. 화면 전환·탭 순회 제거.

---

## 1. 절대 제약 (위반 시 반려)

- **C1. 실현/추정 분리**: Bitget 실현 청산 이력과 Coinglass 추정 모델은 같은 캔버스에 올라오더라도 반드시 별도 레이어 + 별도 라벨. 혼합 집계 금지. 기본 표시는 실현 데이터. 헤더의 "실제 청산 가격 밀집 · 미래 예상 청산대가 아님" 문구는 실현 레이어 활성 시 유지.
- **C2. 표본 수 상시 표기**: N=xxx 표기를 히트맵 레이어 활성 시 항상 노출. N이 임계값(기본 30) 미만이면 "표본 부족" 뱃지.
- **C3. 숫자 조작 금지**: 밀도 스무딩/보간은 렌더링 단계에서만. 집계 원본 값 변형 금지. 툴팁에는 항상 원본 버킷 값 표시.
- **C4. Read-only 불변**: 이 작업은 관측 도구다. 주문/실행 경로에 손대지 않는다.
- **C5. 기존 밴드 뷰 삭제 금지**: 기존 실현 청산 밀집도 카드는 이번 WO에서 제거하지 않는다. 통합 차트 검증 완료 후 별도 WO로 처리.

## 2. Phase 1 — 데이터: 시간×가격 밀도 그리드

### 2.1 집계 파이프라인

청산 이벤트(timestamp, price, size_usd, side)를 2D 그리드로 집계하는 엔드포인트 신설.

```
GET /api/liq/heatmap?symbol=ETHUSDT&tf=4h&from=...&to=...&price_bins=120&side=all
→ {
    time_buckets: [t0, t1, ...],          // tf 캔들 경계와 동일
    price_bins: { min, max, step },        // 뷰포트 가격 범위 기준
    grid: [[v, ...], ...],                 // [time][price] 누적 청산 USD
    n_events: 208,
    side_split: { long_usd, short_usd },
    last_event_ts
  }
```

- 시간 버킷은 **차트 캔들 경계와 동일 규격** (4H 차트면 4H 버킷). 별도 시간축 만들지 말 것.
- 가격 빈은 뷰포트 기준 동적 계산 (기본 120빈, 줌 레벨에 따라 서버 재요청 대신 클라이언트에서 리빈 허용 — 단 원본 이벤트 배열도 함께 내려서 클라이언트 리빈이 원본 기준으로 수행되게).
- **소멸(persistence) 처리**: 실현 이벤트는 발생 시점 버킷에만 기록하는 `event` 모드와, 발생 후 가격이 해당 레벨을 재통과할 때까지 우측으로 지속되는 `persist` 모드 둘 다 지원. 기본값 `persist` (레퍼런스 영상의 "밴드가 이어지다 가격이 닿으면 끊기는" 시각 효과). 지속 계산은 서버에서 수행하고 grid에 반영.

### 2.2 필터 파라미터

- `side`: all / long / short
- `min_size`: 청산 규모 하한 (USD). Coinglass의 레버리지 필터 대응 — 우리 실현 데이터에 레버리지 필드가 없거나 불완전하면 **규모 분위(quartile) 필터로 대체**하고 UI 라벨도 "규모"로. 레버리지 필드가 있으면 10x/25x/50x/100x 그대로.
- `tf` + `range`: 12H / 24H / 3D / 1W / 1M 프리셋

### 2.3 Coinglass 추정 레이어 (선택)

- 이미 사이드바에 Coinglass 추정 모델 칩이 있음. 동일 그리드 포맷으로 `/api/liq/heatmap?source=coinglass_est` 지원. 데이터가 없으면 레이어 칩 비활성 + "미연동" 툴팁. 이번 WO에서 신규 수집기 구축은 범위 외 — 어댑터 인터페이스만 뚫어둘 것.

## 3. Phase 2 — 렌더링: 단일 캔버스 레이어 합성

### 3.1 드로우 순서 (아래→위)

1. 히트맵 래스터 (offscreen canvas에 사전 렌더 → 메인 캔버스 drawImage)
2. 그리드/축
3. 지속 밴드 상위 3개 하이라이트 (현 밴드 뷰의 최대 밀집 강조 이관)
4. 캔들 + 볼륨
5. 플랜 레이어 (익절1/익절2/감시/무효화/내 진입 — 사이드바 가격을 차트 수평선+우측 가격 태그로 상시 표시)
6. EMA 등 지표 레이어
7. 현재가 라인 + 우측 가격 태그 (점선, 실시간)
8. 크로스헤어 + 툴팁

### 3.2 컬러 스케일

- 레퍼런스와 동일 계열: 저밀도 남보라 → 청록 → 녹색 → 고밀도 노랑.
- 스케일은 **로그 정규화** (청산 규모 분포가 롱테일이라 선형이면 노랑이 거의 안 나옴). 뷰포트 내 최대값 기준 상대 스케일, 우측 하단에 컬러 레전드 바 + min/max USD 값 표기.
- 다크 배경에서 캔들 가독성 유지: 히트맵 레이어 전역 opacity 기본 0.55, 설정에서 0.2–0.9 조절.

### 3.3 성능 기준

- 히트맵은 데이터/뷰포트 변경 시에만 offscreen 재렌더. 현재가 라인·크로스헤어는 매 프레임 오버레이만.
- 줌/팬 60fps 유지 (M시리즈 맥 기준). 팬 중에는 offscreen 캔버스를 transform만 하고, 팬 종료 후 리빈+재렌더.
- 뷰포트/시간축은 모든 레이어가 **하나의 상태 객체 공유**. 레이어별 독립 축 금지.

## 4. Phase 3 — UI: 필터 인라인화 + 다중 레이어

### 4.1 레이어 칩 (기존 탭 대체)

- 플랜 / 레벨 / **청산밀집(실현)** / 청산밀집(추정) / 볼륨 / 와이코프 / 하모닉 / EMA / 온체인
- **배타 탭 → 다중 선택 토글**로 전환. 활성 칩은 테두리 강조. shift-클릭 비교(최대 2) 동작은 유지하되, 일반 클릭은 토글 on/off.
- 기본 활성: 플랜 + 청산밀집(실현). 사용자 선택 조합은 localStorage 저장.
- 레이어 4개 이상 동시 활성 시 시각 과부하 — 칩 옆에 레이어별 opacity 슬라이더(hover 시 노출).

### 4.2 상단 필터 바 (히트맵 레이어 활성 시 노출)

```
[전체 | 롱만 | 숏만]  [전체 규모 | Q2+ | Q3+ | Q4]  [12H | 24H | 3D | 1W | 1M]  [persist | event]
```

- 클릭 즉시 리렌더, 화면 전환 없음. 현재 24H/3D 토글과 새로고침 버튼은 이 바로 흡수.

### 4.3 정보 이관

- 기존 밴드 카드의 요약 지표(최대 실현 밀집, 현재가 위/아래 규모, 롱/숏 청산 강도, 마지막 이벤트)는 차트 상단 콤팩트 스트립으로 이관. 클릭 시 해당 가격대로 차트 스크롤+플래시.
- 실현 집중 가격대 3개(1888.85 / 1897.29 / 1901.5 유형)는 3.1의 하이라이트 밴드와 동일 데이터 소스.

## 5. Phase 4 — 라이브 업데이트

- 현재가: 기존 Bitget 라이브 피드 그대로, 라인+태그 1초 이내 반영.
- 히트맵: 신규 청산 이벤트 수신 시 마지막 시간 버킷만 증분 갱신 (전체 재집계 금지). 신규 이벤트는 발생 지점에 1.5초 펄스 마커.
- persist 모드에서 가격이 밴드 통과 시 해당 밴드 우측 끝을 그 시점에서 절단 (소멸 시각화).
- 폴링이면 5초 주기 상한. 웹소켓 경로가 이미 있으면 그걸 사용.
