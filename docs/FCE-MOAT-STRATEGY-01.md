# FCE 해자 전략서 — 레포 실사 기반 (BCG식)

- 판정일: 2026-07-19
- 실사 대상: `mlender-ai/fomo-control-engine` main (WO-FCE-87까지)
- 실사 범위: docs, backend/app, AGENTS.md, WO 이력, 테스트 구조
- 성격: 전략 판정 정본. 근거 없는 강점과 개선 주장을 금지한다.

## 1. Executive Summary

FCE의 해자는 차트·데이터·지표가 아니다. 핵심은 판단 당시 입력과 결론을 원장에 남기고, T+1/T+5/T+20 결과로 강제 채점하며, 적중률 CI 하한을 다음 판단의 가중치에 되먹임하고, 나빠진 신호는 자동 강등하되 좋아지는 방향의 승격은 반드시 사람 승인과 거부권을 거치게 만든 폐루프다.

> 해자의 두께 = 채점된 판단의 수 × 검증 레일의 엄격함

기능 수가 아니라 시간이 쌓인 판단 자본이 해자다. 사생결단의 전선은 폐루프를 두껍게 만드는 것이며 범용 차트 기능 동등화 경쟁에서는 철수한다.

## 2. 전장 정의

### 2.1 킬존

| 전선 | 상대 | 판정 |
|---|---|---|
| 차트 렌더링·레이어·지표 | TradingView | 수백 명과 15년을 기능 수로 따라가는 소모전 |
| 청산 히트맵·전 거래소 파생 데이터 | Coinglass | Bitget 단일 소스로 어그리게이션 경쟁 불가 |
| 시세·뉴스·범용 스크리너 | Toss | 우리가 쓰는 API의 주인과 정면 경쟁 불가 |
| 검증되지 않은 AI 시그널 판매 | 시그널 판매상 | 정직성 규범과 충돌 |

차트·UI WO는 장애, 가독 불능, 데이터 오표시를 막는 위생 작업일 때만 허용한다. 위생 필요성을 쓰지 못하면 착수하지 않는다.

### 2.2 승부처

FCE 한 인스턴스만 동시에 가진 교집합은 다음 네 축이다.

- Bitget 실행 데이터: 실제 체결·포지션
- Toss 기초자산 구조·수급·리테일 주목도
- 사용자의 판단 전 이력과 사후 채점
- 통계 레일과 비대칭 자율 상태머신

Bitget/Toss 어댑터는 복제할 수 있다. 과거의 사용자 판단은 시간을 되돌리지 않는 한 복제할 수 없고, 정직한 자기 채점은 거래 회전율이나 시그널 판매와 이해가 충돌하는 경쟁자가 구조적으로 운영하기 어렵다.

## 6. 구조적 해자

| 경쟁자 | 수익 모델 | 정직한 자기 채점기와의 충돌 |
|---|---|---|
| 증권사·거래소 | 거래 수수료·주문 흐름 | 회전율 감소와 투자권유 규제 위험 |
| 시그널 판매상 | 구독 | CI 하한과 실패 공개가 상품성과 충돌 |
| 차트 플랫폼 | 구독·광고 | 사용자 행동·판단 이력 부재 |
| 데이터 어그리게이터 | 광고·API | 개인 판단 축 부재 |

FCE는 사용자와 이해가 충돌하는 수익 모델이 없는 위치에서 자기 판단을 강제로 채점할 수 있다.

## 8. 3-Horizon 전략

### H1 — 폐루프 완결

신규 기능을 동결하고 채점 안 되는 판단을 없앤다.

- WO-FCE-88: 실제 히스토리 stance 백테스트.
- WO-FCE-89: 원장 커버리지 전수화.
- WO-FCE-90: Shadow Account v2.
- WO-FCE-91: Dogs 철거.

### H2 — 검증 공장 실적

validated 승격 1호와 유의미한 Improvement Proof 1회를 만든다. 교차 신호를 상태머신에 태우고 고래 28일 검증을 판정한다.

### H3 — FOMO Club 이전

H1·H2에서 실증된 원장 스키마, 통계 레일, 비대칭 자율만 양산 환경으로 이전한다. FCE는 실험실이고 FOMO Club은 양산 환경이다.

## 상시 판정 질문

1. 이 작업은 채점되는 판단을 늘리는가?
2. 통계와 검증 폐루프를 더 엄격하게 만드는가?
3. 경쟁사가 6개월 안에 복제할 수 있는 범용 기능인가?
4. 차트·UI 위생 작업이라면 장애·오판 방지 근거가 있는가?

판단이 채점되지 않으면 해자에 쌓이지 않는다. 신규 기능보다 실성적, 원장 커버리지, FOMO 귀속을 우선한다.
