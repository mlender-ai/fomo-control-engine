## 킬존 대조와 위생 필요성
- `docs/FCE-MOAT-STRATEGY-01.md` §2.1의 범용 차트·지표 확장이 아니다.
- 원본 원장에는 `flip`과 양방향 전환 순서가 기록돼 있지만 텔레그램·대시보드가 최종 방향만 표시해 동시 양방향 보유처럼 오인시킨다.
- 최근 이벤트는 한 지갑·한 종목의 분할 체결이 상위 10칸을 독점해 다른 확정 체결이 존재하지 않는 것처럼 보인다.
- 순체결 흐름은 서로 다른 종목을 합산하면서 종목별 검증 경로가 없어 데이터 검증이 불가능하다. 따라서 오표시·가독 불능을 막는 허용된 위생 작업이다.

## 진단 (코드 확정)
- `backend/app/notify/alerts.py`는 `open`과 `flip`을 모두 최종 `side`의 “롱/숏 관측”으로 출력한다. 실제 #151 원장은 `2026-07-20 06:33 UTC` 숏→롱, `18:12 UTC` 롱→숏 전환이다.
- `backend/app/onchain/service.py`는 최신 원본 100건을 그대로 내려주고 UI는 첫 10건만 표시한다. 같은 SNDK 계정의 초단기 분할 체결이 목록을 독점한다.
- `event.symbol`이 비어 있는 `XYZ:SNDK`, `@107` 같은 Hyperliquid 원시 코인은 흐름 집계에는 들어가지만 종목 키가 없어 필터 대상에서 누락된다.
- 검증 상태와 표본 N은 이벤트 payload에 있으나 최근 체결 UI에서 숨겨져 있다.

## 작업
### 1. 전환 의미 보존
- `flip long`은 “숏→롱 전환”, `flip short`는 “롱→숏 전환”으로 텔레그램과 화면에서 명시한다.

### 2. 원장 보존형 체결 묶음
- 원본 `whale_events`는 수정·삭제하지 않는다.
- 같은 지갑·종목·이벤트·방향의 60초 이내 분할 체결만 표시 계층에서 합산하고 체결 수와 원본 ID를 남긴다.
- 종목별 라운드로빈으로 최근 목록을 구성해 한 종목이 전체 목록을 독점하지 못하게 한다.

### 3. 종목별 순체결 검증
- 정규 `symbol`이 없으면 원시 `coin`을 `instrument` 키로 사용한다.
- 전체 흐름과 동일한 정의로 종목별 72시간 타임라인·24시간 체결·현재 노출을 제공한다.
- 대시보드에서 검색·선택하고 종목 쏠림 행을 눌러 동일 필터를 적용한다.

### 4. 미검증 상태 가시화
- 최근 체결에 지갑명, 검증 상태, N, 분할 체결 수를 표시한다.
- 미검증 고래는 관측 데이터로만 유지하고 컨플루언스·자동 진입 경로는 변경하지 않는다.
