# WO-FCE-TOSS-PAPER-02 — 주식 페이퍼 진입 파이프라인 개통

- 우선순위: P0
- 대상: Toss 주식 스카우트 · 주식 PaperBroker · 엔진 트레이딩
- 상태: 구현

## 문제와 범위

주식 트랙의 4주 검증이 인증 401, 전시장 랭킹과 거래 유니버스 혼합, 검증 수단이 검증 완료를 선행 요구하는 순환 게이트 때문에 시작되지 않았다. 이 WO는 게이트를 낮추지 않고 세 병목을 각각 진단·분리·공용 분석 연결로 해소한다. 실주문 경로는 만들거나 변경하지 않는다.

## 구현 규범

1. 인증: 토큰/시크릿을 노출하지 않는 단계별 진단 API와 운영 런북을 제공한다.
2. 유니버스: 전시장 랭킹은 관측으로 유지하되 `tradable`과 `role`을 기록하고, 진입은 200종목 정본만 허용한다. QQQ/237350은 `benchmark_proxy`다.
3. 분석: `signals.py`에 무효화선/RR을 추가하지 않는다. 저장된 Toss 캔들을 `build_chart_analysis`·`build_analyst_briefing`·`generate_report` 공용 엔진에 넣는다.
4. 정책: `stock-v1`을 보존하고 `stock-v2`를 추가한다. 시그니처는 기록 전용, 실적 게이트는 `not_evaluable`이다. RR·무효화선·확정 flip·증거·체크리스트·점수·신선도는 필수로 유지한다.
5. 시계: 인증 후 첫 정상 관측 시점에만 시장별 4주 시계를 시작한다. 고장 구간은 무효 사유를 보존한다.
6. 원장: 모든 실패 게이트의 측정값과 임계를 append-only 거부 원장에 기록하고 최근 7일 분포를 표시한다.

## 킬존 대조

신규 TA/차트 기능이 아니다. 인증 장애·거래 대상 오표시·검증 시계 오염을 제거하고 판단 원장의 입력 정직성을 회복하는 P0 위생 작업이다. 신규 감지기를 추가하지 않는다.

## 수용 기준

- 단계별 인증 진단과 `TossAuthRunbook.md`
- 관측 전용 후보가 스카우트에는 남고 PaperBroker 후보에서는 제외
- 저장된 실캔들로 공용 구조/합류 분석 실행
- v1/v2 diff와 시그니처 순환 봉쇄 제거 증빙
- 인증 성공 + 첫 정상 수집에서만 시계 시작
- 최근 7일 게이트별 거부 원장 노출
- 실주문 코드/경로 변경 0건
