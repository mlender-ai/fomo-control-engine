## 진단 (코드 확정)

- 실제 BTCUSDT 96시간 캐시에서 약 440만 행을 읽고 Pydantic JSON 검증과 체결 집계를 반복해 차트 분석 한 번에 35~69초가 걸렸다.
- 캐시/JSON 손상과 SQLite 읽기 오류가 차트 요청의 처리되지 않은 500으로 전파될 수 있다.
- 기존 coverage는 API 페이지 제한이나 캐시 표본 제한과 관계없이 요청한 96시간 전체를 관측한 것처럼 기록한다.

## 작업
### 1. 실체결 캐시 읽기 상한
- 최신 체결만 인덱스로 읽는 bounded query를 추가하고 기본 상한을 설정으로 노출한다.

### 2. 실패 격리와 정직한 coverage
- SQLite/저장 JSON 오류는 API 재조회 또는 데이터 없음 fallback으로 격리한다.
- 실제 첫·마지막 체결 시각, 요청 범위, 제한 여부를 분리해 누락 범위를 합성하지 않는다.

### 3. 회귀 검증
- 최신 N건의 시간순 반환, 제한 메타데이터, 손상 데이터 fallback을 테스트한다.
- 문제 포지션의 실제 차트 API 응답 시간과 브라우저 복구를 확인한다.
