# Benchmark Liveness

## User account source

- 자금흐름의 `get_trade_fills`는 공개 시장 체결(`/api/v2/mix/market/fills-history`)이며 사용자 거래가 아니다.
- 대결의 "나"는 인증된 계정 체결(`/api/v2/mix/order/fills`)만 사용한다.
- `sync_user_fills`는 첫 실행에 최대 89일을 수집하고 이후 마지막 체결에서 24시간을 겹쳐 재수집한다.
- 원본은 `user_account_fills`, 종료 포지션 재구성 결과는 `user_trades`에 영구 보존한다.
- 조회 범위 안에서 완결된 재구성 거래는 모두 저장한다. 최근 28일 참고 성과와 대결 시작 시각 이후 판정 성과는 별도 창으로 계산한다.
- 실계좌 현재 오픈 포지션과 재구성 잔여 포지션의 심볼·방향·수량을 매 동기화마다 대조한다.

## Reconstruction limits

- 복수 진입은 잔여 수량의 가중 원가로 계산하고 부분청산은 같은 포지션 생명주기에 합산한다.
- 헤지 모드는 `tradeSide`와 매수/매도로 롱·숏을 분리한다. 원웨이 모드 반전 주문은 기존 반대 포지션을 먼저 닫고 남은 수량을 새 방향 진입으로 처리한다.
- 순손익은 체결 가격 손익에서 USDT 표시 수수료를 차감한 **체결 기반 재구성값**이다.
- BGB 등 다른 코인으로 낸 수수료의 USDT 환산값은 API 응답에 없으므로 차감하지 않는다.
- `profit` 응답값은 감사용으로 별도 저장하지만 화면의 재구성 손익을 거래소 확정 손익으로 표기하지 않는다.
- 최대 조회 범위보다 오래 열린 포지션의 진입 체결이 없으면 해당 청산은 `unmatched_close_fills`로 남고 대결 거래로 만들지 않는다.

## Candidate bootstrap

- 일반 경로는 승인된 `validated` 시그니처와 CI 하한 게이트를 그대로 사용한다.
- 초기 공백에서만 최신 확정 캔들에 발생한 candidate가 백테스트 `N >= 15`, `win@1R >= 50%`를 충족하면 `candidate_bootstrap`으로 허용한다.
- 이 경로는 퍼널, 진입 근거, 시그니처 스냅샷에 태깅한다.
- 승인된 validated 시그니처가 3개 이상이거나 4주 대결 창이 끝나면 자동 비활성화된다.
