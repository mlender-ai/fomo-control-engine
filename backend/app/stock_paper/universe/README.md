# 주식 페이퍼 유니버스 갱신

`2026-q3.json`은 런타임 API 검색 결과가 아니라 버전 관리되는 수동 시드다. 페이퍼 포지션의 재현성을 위해 실행 중 파일을 덮어쓰지 않는다.

분기 갱신 절차:

1. Nasdaq 공식 Nasdaq-100 구성종목과 KRX 정보데이터시스템의 코스피100 구성종목을 확인한다.
2. 새 `YYYY-qN.json` 파일을 추가하고 `version`, `effective_at`, `source_as_of`를 갱신한다.
3. 시장별 고유 심볼 100개, 합계 200개인지 `test_stock_paper_execution.py`로 검증한다.
4. 편출 종목은 새 파일에서 제거한다. 기존 페이퍼 포지션은 유지하고 `StockUniverse.entry_allowed`가 신규 진입만 막는다.
5. 리뷰에서 변경 목록과 출처를 승인한 뒤 기본 파일 경로를 새 버전으로 전환한다.

Nasdaq의 GOOG/GOOGL처럼 한 회사가 복수 주식 클래스로 지수에 포함되면 이 페이퍼 유니버스는 회사 단위 100개를 유지한다. 현재 시드는 GOOGL만 포함하며 이 선택은 JSON `notes`에 기록돼 있다.

경고 상태는 정적 파일에 복사하지 않는다. Toss `warnings`의 정리매매·투자위험·VI·거래정지를 매 수집 시 런타임 하드 게이트로 적용한다.
